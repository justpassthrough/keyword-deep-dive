# 키워드 딥다이브 스캐너

약사 블로그 글감을 자동으로 찾아주는 도구입니다.
"마운자로"처럼 하나의 **뿌리 키워드**에서 "마운자로 부작용", "마운자로 품절" 같은 **복합 키워드**를 자동으로 확장하고, 각각의 **약사 블로그 글감 가치**를 점수로 매깁니다.

**대시보드**: [https://justpassthrough.github.io/keyword-deep-dive/](https://justpassthrough.github.io/keyword-deep-dive/)

---

## 전체 구조 한눈에 보기

```
뿌리 키워드 6개 (마운자로, 알부민, 위고비, 오메가3, NMN, 밀크씨슬)
       │
       ▼
 [dive.py] 복합키워드 확장 + 분석
       │
       ├── 네이버 블로그/뉴스 제목에서 복합키워드 추출
       ├── DataLab 검색량 비교
       ├── 약사 가치 점수 산출
       └── 결과 → data/latest.json
       │
       ▼
 [build_dashboard.py] JSON → HTML 대시보드
       │
       └── docs/index.html (GitHub Pages로 공개)
```

---

## 파일 설명

| 파일 | 하는 일 |
|------|---------|
| `scripts/dive.py` | **메인 스크립트**. 뿌리 키워드마다 복합키워드를 찾고, 검색량/변화율/전문가갭/약사가치를 분석합니다. |
| `scripts/build_dashboard.py` | `data/latest.json`을 읽어서 보기 좋은 HTML 대시보드(`docs/index.html`)를 만듭니다. |
| `scripts/discover_roots.py` | 새로운 뿌리 키워드를 자동으로 발굴합니다. 매주 월요일 실행됩니다. |
| `data/root_keywords.json` | 현재 관리 중인 뿌리 키워드 목록입니다. |
| `data/known_products.json` | 알려진 제품/성분 이름 사전입니다. |
| `data/latest.json` | 가장 최근 분석 결과입니다. |
| `docs/index.html` | 대시보드 웹페이지입니다. |

## 자동 실행 스케줄

GitHub Actions가 자동으로 실행합니다.

| 워크플로우 | 실행 시점 | 하는 일 |
|-----------|----------|---------|
| `daily_dive.yml` | **매일 오전 8시, 오후 8시** (한국시간) | dive.py → build_dashboard.py → 결과 커밋 |
| `weekly_discover.yml` | **매주 월요일 오전 8시** (한국시간) | discover_roots.py → dive.py → build_dashboard.py → 결과 커밋 |

수동으로도 실행 가능합니다:
- GitHub 레포 → Actions 탭 → 원하는 워크플로우 → "Run workflow" 버튼

## 대시보드 읽는 법

### 추천 글감 TOP 3
모든 뿌리를 통합해서 **약사 가치가 가장 높은** 복합키워드 3개를 보여줍니다.

### 뿌리별 탭
각 뿌리 키워드를 클릭하면 해당 뿌리의 복합키워드 목록이 나옵니다.

**라벨 의미:**
| 라벨 | 의미 |
|------|------|
| ★★★★ | 약사 전문성이 매우 필요한 키워드 (부작용, 상호작용, 복용법 등) |
| ★★★ | 약사 관점이 유용한 키워드 (효과, 품절, 대체 등) |
| ★★ | 경험담 위주 키워드 (후기, 전후 등) |
| ★ | 구매/접근성 키워드 (가격, 약국 등) |
| 🔥급등 | 최근 7일 검색량이 20% 이상 증가 |
| 🔗비교형 | 다른 뿌리 키워드를 함께 포함 (예: "마운자로 위고비 차이") |
| 📎롱테일 | DataLab에는 안 잡히지만 블로그에서 수요가 있는 키워드 |

**전문가 갭:**
- **전문가 갭 큼**: 블로그는 많은데 약사 콘텐츠가 매우 부족 → **글 쓰기 좋은 기회**
- **전문가 부족**: 약사 콘텐츠가 부족한 편
- **보통**: 적당히 있음
- **전문가 포화**: 약사 콘텐츠가 이미 많음 → 경쟁 심함

### 미확인 후보 (🔍)
분석 중 자주 등장했지만 `known_products.json` 사전에 없는 단어입니다.
제품이나 성분 이름이 맞다면 사전에 추가하면 됩니다.

## 키워드 추가/관리 방법

### 뿌리 키워드 수동 추가

`data/root_keywords.json` 파일을 열어서 `roots` 배열에 새 항목을 추가합니다.

```json
{
  "keyword": "글루타치온",
  "category": "영양제",
  "status": "active",
  "source": "manual",
  "parent": null,
  "added": "2026-03-04",
  "last_active": "2026-03-04",
  "consecutive_dormant_weeks": 0
}
```

- **keyword**: 추가할 키워드
- **category**: 분류 (비만치료제, 영양제, 일반의약품 등 자유 입력)
- **status**: `active`로 설정하면 바로 분석 대상
- **added**: 추가한 날짜

### 뿌리 키워드 자동 발굴

`discover_roots.py`가 매주 자동으로 후보를 찾습니다.

**4가지 소스:**
1. **이웃 발견**: 현재 active 뿌리 블로그에서 다른 제품명 추출
2. **카테고리 시드**: "비만치료제 신약", "영양제 인기 순위" 등 검색
3. **약업계 뉴스**: "의약품 허가", "건강기능식품 출시" 등 뉴스 검색
4. **트렌드 스캐너 연동**: health-trend-scanner 결과에서 제품명 추출

후보가 발견되면 `watch` 상태로 등록 → 2회 연속 유의미하면 `active`로 승격됩니다.

### 제품 사전 업데이트

`data/known_products.json`의 `products` 배열에 새 제품/성분명을 추가하면 됩니다.
대시보드의 "미확인 후보" 섹션에서 자주 보이는 단어가 있으면 여기에 추가해 주세요.

## 뿌리 키워드 생명주기

```
[watch] ──(2회 유의미)──→ [active] ──(4주 급등 0)──→ [dormant] ──(8주)──→ [archived]
                              ↑                           │
                              └───(급등 재등장)────────────┘
```

- **active**: 매일 2회 분석 대상
- **watch**: 관찰 중 (자동 발굴된 후보)
- **dormant**: 4주 연속 급등 키워드가 없으면 잠시 쉼
- **archived**: 8주 이상 dormant → 분석에서 제외

## 로컬에서 직접 실행하기

```bash
# 프로젝트 폴더로 이동
cd keyword-deep-dive

# 환경변수 설정
export NAVER_CLIENT_ID=발급받은_ID
export NAVER_CLIENT_SECRET=발급받은_SECRET

# 분석 실행
python scripts/dive.py

# 대시보드 생성
python scripts/build_dashboard.py

# docs/index.html을 브라우저로 열면 대시보드 확인 가능
```

## 약사 가치 점수 산출 방식

```
약사 가치 = 의도 점수 × 전문가 갭 배수 × 변화율 보너스
```

- **의도 점수** (1~4): 키워드에 포함된 단어로 판단
  - 4점: 부작용, 상호작용, 복용, 금기, 용량, 처방, 성분, 원리, 차이, 비교 등
  - 3점: 효과, 감량, 품절, 대체, 추천, 논란 등
  - 2점: 후기, 경험, 결과, 전후 등
  - 1점: 가격, 최저가, 할인, 구매, 택배 등

- **전문가 갭 배수**: 전체 블로그 대비 "약사" 포함 블로그 비율
  - ×1.3: 전문가 갭 큼 (비율 30배 이상)
  - ×1.1: 전문가 부족
  - ×1.0: 보통
  - ×0.7: 전문가 포화

- **변화율 보너스**: 최근 검색량 급등 시 가산
  - ×1.2: 변화율 +20% 이상
  - ×1.4: 변화율 +50% 이상

## API 키 정보

네이버 API 키가 2개 세트 있습니다 (기존 트렌드 스캐너와 분리):

| 용도 | GitHub Secret 이름 |
|------|-------------------|
| 딥다이브 스캐너 전용 | `NAVER_CLIENT_ID_DIVE` / `NAVER_CLIENT_SECRET_DIVE` |
| 트렌드 스캐너 전용 | `NAVER_CLIENT_ID` / `NAVER_CLIENT_SECRET` |

DataLab API는 하루 1,000건 제한이 있어서 키를 분리했습니다.
